# Tutorial: Editor Markdown + Git (sem terminal)

Este guia mostra um fluxo simples para editar suas notas em Markdown e publicar sem usar o terminal.

## Objetivo

- Editar notas com seu editor favorito.
- Sincronizar com o GitHub usando interface grafica.
- Ver o site atualizado automaticamente no GitHub Pages.

## Fluxo recomendado (sem terminal)

1. Instale o GitHub Desktop.
2. Crie um repositorio a partir do template (ou clone o seu repositorio).
3. Abra a pasta local no seu editor.
4. Edite as notas em src/pages.
5. Use o GitHub Desktop para Commit e Push.

## Editores suportados (exemplos)

### VS Code

- Abra a pasta do repositorio.
- Edite arquivos .md em src/pages.
- Use o painel Source Control para ver mudancas.

### Obsidian

- Abra a pasta do repositorio como vault.
- Crie e edite notas em src/pages.
- Use o GitHub Desktop para Commit e Push.

### iA Writer

- Abra a pasta do repositorio.
- Edite os arquivos .md em src/pages.
- Use o GitHub Desktop para Commit e Push.

## Checklist de publicacao

- [ ] Editei um arquivo em src/pages.
- [ ] Adicionei imagens em src/assets (se necessario).
- [ ] Fiz Commit com uma mensagem curta.
- [ ] Fiz Push para o GitHub.
- [ ] Aguardei o deploy do GitHub Pages.

## Dicas rapidas

- Arquivos iniciados com _ sao ignorados.
- Cada pagina precisa de frontmatter com layout e title.
- Imagens devem estar em src/assets para serem copiadas.
